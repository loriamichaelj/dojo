# Overview

Notes, reviews, and reflections on books — what was read, what stuck, and why it
matters. Each entry is a single markdown file; lead with a one-line takeaway so it
stays scannable later.

## Shelves

| Folder | Scope |
|---|---|
| [`fiction/`](./fiction/) | Novels, short stories, and literary works |
| [`non-fiction/`](./non-fiction/) | Quant, finance, technical, biography, and everything else grounded in fact |

## How to use this folder

- One shelf per subfolder; create a new `.md` file per book.
- Suggested file naming: `author-title-slug.md`, or `YYYY-MM-DD-slug.md` for dated notes.

## Book file structure

Each book file follows the same structure: the **title** as the H1, then
**author**, **publish date**, and a **synopsis** section.

```markdown
# Title

| | |
|---|---|
| **Author** | First Last |
| **Published** | YYYY |

## Synopsis

A few sentences summarizing what the book is about.
```

The H1 title doubles as the page's title and nav label. Use a four-digit year
for **Published** (add month/day if useful, e.g. `YYYY-MM-DD`).
